# 15. Documentation & User Manuals Workflow

Quy trình "đóng gói tri thức" để đảm bảo dự án có thể được bảo trì và sử dụng lâu dài bởi bất kỳ ai.

### Bước 1: Technical Documentation
- [ ] **Viết tài liệu Kiến trúc (Technical Specs)**
  - `@Tech_Lead` và `@Senior_Dev` cập nhật tài liệu về sơ đồ Database, API Endpoints, và các Logic nghiệp vụ đặc biệt.
  - *Lệnh mẫu:* `@Senior_Dev /write Hãy soạn thảo tài liệu API cho module Quản lý Tài sản cố định.`

### Bước 2: User Guides (Hướng dẫn sử dụng)
- [ ] **Soạn thảo Manual cho End-user**
  - `@BA` viết hướng dẫn từng bước cho các phòng ban (Kế toán, Kho, Nhân sự) dưới dạng Wiki hoặc PDF.
- [ ] **Quay video hướng dẫn (Video Tutorial)**
  - `@Designer` hoặc `@BA` thực hiện ghi màn hình hướng dẫn các tính năng phức tạp.

### Bước 3: Quản lý Phiên bản (Version Control)
- [ ] **Cập nhật Changelog**
  - `@Managing_Director_PM` liệt kê các thay đổi trong từng phiên bản release.

### Bước 4: Hệ thống Knowledge Base (KB)
- [ ] **Xây dựng Wiki nội bộ**
  - Đưa toàn bộ tài liệu lên công cụ quản lý chung để mọi người dễ dàng tra cứu.
